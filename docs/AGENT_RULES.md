# Odin Engineering Agent Rules

You are an autonomous software engineer working on Odin.

Always:

- Read the repository before making changes.
- Preserve the existing architecture.
- Explain your implementation plan before large changes.
- Prefer extension over replacement.
- Create focused commits.
- Never commit broken code.
- Keep documentation current.
- Add or update tests when behavior changes.

When implementing a milestone:

1. Audit.
2. Plan.
3. Implement.
4. Validate.
5. Test.
6. Summarize.

If multiple solutions exist, choose the one that best supports Odin’s long-term architecture.

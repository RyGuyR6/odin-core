# OIC-016 — Autonomous Git

## Goal

Allow Odin to publish validated engineering work through a durable,
approval-gated Git workflow without permitting direct autonomous changes to
protected branches.

## Safety contract

- Every operation is bound to a workspace and immutable expected HEAD SHA.
- `main`, `master`, production, and release branches cannot be autonomous
  mutation targets.
- Commits require a passing validation result bound to the pre-commit HEAD.
- Push and pull-request creation require OIC-014 execution approval.
- Pull requests created by Odin are always drafts.
- Merge readiness requires approvals, no requested changes, mergeability, and
  successful checks.
- OIC-016 does not autonomously merge pull requests or publish releases.

## Execution handlers

- `git.branch`
- `git.commit`
- `git.push`
- `git.pull_request`
- `git.readiness`
- `git.release_plan`

Remote mutation handlers participate in OIC-014 mutation receipts, preventing
blind replay after an uncertain worker crash.

Release planning produces a validated immutable manifest only; tag and release
publication remain explicit approval-gated future mutations.

## Acceptance workflow

Analyze → workspace → implement → validate → branch → commit → approve → push
→ approve → draft PR → evaluate readiness.

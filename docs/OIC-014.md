## Status

The durable backend execution domain is implemented. Repository mutation handlers remain
approval-gated adapters: OIC-014 supplies the lifecycle, queue, attempts, approvals,
recovery, events, and API contracts within which those handlers execute.

## Architecture

- `app.execution.models`: runs, steps, attempts, approvals, and limits.
- `app.execution.persistence`: SQLite authority, atomic queue claims, leases, and recovery.
- `app.execution.controller`: dependencies, attempts, retries, approvals, and completion.
- `app.execution.service`: create, inspect, resume, approve, reject, and cancel operations.
- `app.execution.worker`: cooperative durable worker.
- `app.execution.handlers`: fail-closed typed handler registry.
- `app.api.executions`: authenticated HTTP API.

Persisted state is authoritative. Events are an observability projection.

## Queue, recovery, and idempotency

- Claims are selected and updated in one SQLite transaction.
- Claims include worker identity and lease expiration.
- Active leases cannot be claimed by another worker.
- Expired work is marked interrupted, released, reported, and safely re-queued.
- Queue uniqueness prevents duplicate run/step entries.
- Idempotency keys are unique within a run.

External side-effect handlers must use the step idempotency key and persist the external
operation or artifact identifier. Queue delivery is at-least-once after a crash.

## Retries and approvals

Failures use bounded exponential backoff persisted as `available_at`.
`NonRetryableExecutionError` fails immediately.

A step with `requires_approval` pauses before its handler executes. Decisions record the
exact run and step, reviewer, reason, and timestamp. Rejection cancels without executing
the handler. Workspace mutation handlers must also verify `TaskWorkspaceService` proposal
approval; execution approval does not replace workspace approval.

## Resource limits

Runs persist limits for attempts, steps, tool calls, runtime, and cost. The core enforces
step and attempt limits. LLM and tool adapters must report and enforce runtime, token,
tool-call, and cost consumption before unattended live autonomy is enabled.

## Compatibility

`/change-tasks` remains unchanged. New autonomous workflows use `/executions`.
Migration translates `ChangeStep.action` values into execution handlers while all
repository writes continue through `TaskWorkspaceService`.
